# License Decision

## Selected Default

This repository currently uses `MIT`.

## Why MIT

- you want the lowest-friction reuse path
- you care more about adoption than about patent language
- the repo is mainly docs, playbooks, and small tooling

## When To Reconsider

- you want explicit patent grants
- you expect companies to evaluate the repo more formally
- you want a slightly stronger legal framing for code reuse

If those concerns become primary, Apache-2.0 is the likeliest next option.

## Before Publishing

- confirm that no copied internal-only assets remain
- confirm that all extracted code is intended for public reuse
- confirm that the repo name, description, and scope are stable enough to publish
