# v0.2 Execution Checklist

Historical note: this checklist defined the tool-extraction work that was ultimately shipped in the `v0.3.0` tag instead of a separate `v0.2.0` release.

This document turns the public roadmap into an execution-ready checklist for the `v0.2` cycle.

## Goal

Move the repository from a docs-first public beta into a more validated and repeatable release by strengthening:

- skill usage evidence
- repository-level validation and contributor workflows
- the forward-tested confidence of `tools/wechat-gui-check`

## Definition of Done

`v0.2` is ready when all of the following are true:

- every active skill has example prompts and at least one recorded prompt validation pass
- the skill validator checks meaningful content requirements, not only file presence
- contributors can run a single local validation entry point
- CI covers skill content, docs hygiene, and tool smoke checks
- `tools/wechat-gui-check` has at least one documented forward-test on a public miniapp repo outside this repository
- release notes and changelog updates are ready for tagging

## Suggested Issue Breakdown

If you want to turn this into GitHub issues, these are good issue-sized units:

1. add example prompts for all four skills
2. add skill review checklist and validation log
3. upgrade `scripts/validate_skills.py`
4. add a single local validation entry point and document it
5. expand CI coverage around docs and examples
6. document external forward-testing for `wechat-gui-check`
7. standardize `report.json` failure classification
8. improve `tools/wechat-gui-check/README.md`
9. prepare changelog, roadmap, and the release notes

---

## Recommended Execution Order

1. finish M1 first so the repository has real usage evidence
2. implement M2 next so the new expectations become enforceable
3. complete M3 once the validation baseline is stable
4. close with M4 so the release story matches the repository state
